fix: add explicit tenant migrations and blocking verification

Problem:
- Production error: "relation finance_invoice does not exist"
- Root cause: Bootstrap commands relied on unreliable auto_create_schema
- Finance app migrations weren't applied to demo tenant schema

Solution:
- Add explicit migrate_schemas calls to all tenant creation methods
- Implement automatic tenant rollback on migration failure
- Add blocking migration verification to Docker entrypoint
- Prevent silent failures with exit 1 on migration errors

Changes:
1. tenants/services.py (lines 117-140)
   - Added explicit migration call to create_tenant()
   - Automatic cleanup on failure

2. tenants/management/commands/bootstrap_demo_tenant.py (lines 384-411)
   - Added explicit migration call after tenant creation
   - CommandError with rollback on failure

3. tenants/management/commands/bootstrap_demo_tenants.py (lines 310-338)
   - Added explicit migration call after tenant creation
   - Per-tenant error handling with cleanup

4. docker/entrypoint.sh (lines 509-558)
   - Added Step 4.5: Demo tenant migration verification (blocking)
   - Added Step 4.6: All tenants migration verification (blocking)
   - Container exits with code 1 if migrations fail
   - Clear error messages with action steps
